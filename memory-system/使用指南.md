# 记忆系统使用指南

## 快速启动

### 方式一：每次对话开始时加载记忆

在对话开始时发送以下指令：

```
/load-memory
```

或手动发送：

```
请读取我的记忆系统（[你的记忆系统路径]/），加载以下内容：
1. Intent/目标与规划.yaml - 了解我的目标
2. Intent/偏好与要求.yaml - 了解我的偏好
3. Intent/约束与边界.yaml - 了解我的边界
4. Memory/L1_情境层.yaml - 了解近期上下文
5. Memory/L2_行为层.yaml - 了解我的习惯

基于以上记忆，调整你的回答方式。
```

### 方式二：需要决策支持时

```
/decision [你的问题]

例如：/decision 我应该选择哪个技术栈？
```

或：

```
请基于我的记忆系统帮我分析这个决策：
[你的问题]

参考：
- Memory/L3_认知层.yaml - 我的思维模式
- Memory/L4_核心层.yaml - 我的价值观
- Intent/目标与规划.yaml - 我的目标
```

## 日常使用流程

### 1. 决策支持

**触发条件**：需要做选择、判断、规划时

**指令**：
```
/decision [问题]
```

**AI 行为**：
1. 读取 L3 认知层、L4 核心层
2. 读取 Intent 目标与约束
3. 基于你的价值观和思维模式给出建议
4. 解释建议背后的逻辑（为什么适合你）

**示例**：
```
/decision 我应该花时间学习 Rust 还是继续深入 Python？

AI 会参考：
- 你的学习偏好（L2 行为层）
- 你的目标方向（Intent/目标与规划）
- 你的决策原则（L4 核心层）
- 你的时间模式（L2 行为层）
```

---

### 2. 每日复盘

**触发条件**：每天结束时

**指令**：
```
/daily-review
```

**AI 行为**：
1. 分析今天的所有对话
2. 识别关键事件、决策、偏好表达
3. 发现行为模式和思维模式
4. 生成每日总结报告
5. 更新 L1 情境层
6. 将候选洞察加入洞察队列

**输出内容**：
- 今日事件摘要
- 发现的行为模式
- 发现的偏好线索
- 候选洞察（待沉淀到更深层）
- 明日建议

---

### 3. 每周复盘

**触发条件**：每周结束时

**指令**：
```
/weekly-review
```

**AI 行为**：
1. 汇总本周所有每日复盘
2. 分析行为趋势和模式
3. 识别重复出现 3 次以上的习惯
4. 生成周报草稿
5. 更新 L1 → L2 沉淀候选
6. 生成下周建议

**输出内容**：
- 本周成就与挑战
- 确认的行为习惯
- 待沉淀的认知模式
- 目标进展评估
- 下周优先事项

---

## 记忆沉淀规则

### L0 → L1（对话 → 情境）
- 会话结束时自动生成摘要
- 重要事件即时记录
- 保留 30 天

### L1 → L2（情境 → 行为）
- 行为模式出现 3 次
- 周/月复盘确认

### L2 → L3（行为 → 认知）
- 发现稳定思维模式
- 季度复盘确认

### L3 → L4（认知 → 核心）
- 年度深度反思
- **仅人工修改**

## 指令速查表

| 指令 | 用途 | 读取层级 |
|------|------|----------|
| `/load-memory` | 加载记忆 | L1, L2, Intent |
| `/decision [问题]` | 决策支持 | L3, L4, Intent |
| `/daily-review` | 每日复盘 | L0 → L1 |
| `/weekly-review` | 每周复盘 | L1 → L2 |
| `/monthly-review` | 月度复盘 | L2 → L3 |
| `/update-intent` | 更新意图 | Intent |
| `/show-memory` | 查看记忆摘要 | 全部 |

## 手动更新记忆

### 更新目标与规划
```
/update-intent goal [目标内容]
```

### 更新偏好
```
/update-intent preference [偏好内容]
```

### 更新约束
```
/update-intent constraint [约束内容]
```

### 更新核心层（需人工确认）
```
/update-core [内容]
```
