# 今日目标对齐・脑倒空优先级规划 + 每日复盘

## 核心作用

1. **清空大脑杂乱待办**，杜绝「想得多做得乱」
2. **绑定长期目标/记忆文档**，自动做优先级排序，避免偏离主线
3. **拆分人工必做 vs Agent自动执行** 两类任务
4. **支持多Agent并行跑任务**，一人顶一个小团队
5. **自动保存规划与复盘**，形成每日记录

---

## 文件存储

**存储目录**：`[你的项目路径]/daily-records/`

| 文件类型 | 命名规则 | 示例 |
|----------|----------|------|
| 每日规划 | `今日规划_YYYY-MM-DD.md` | `今日规划_2026-03-25.md` |
| 每日复盘 | `今日复盘_YYYY-MM-DD.md` | `今日复盘_2026-03-25.md` |

## 模式一：今日规划

### 触发方式
- 说「开始今日规划」
- 说「/daily-plan」
- 说「今日规划」
- 说「脑倒空」

### Step 1: 加载记忆系统

**必须首先读取以下文件**：

```
[你的项目路径]/memory-system/Intent/目标与规划.yaml      # 长期目标
[你的项目路径]/memory-system/Intent/偏好与要求.yaml      # 交互偏好
[你的项目路径]/memory-system/Intent/约束与边界.yaml      # 决策边界
[你的项目路径]/memory-system/Memory/L2_行为层.yaml       # 工作习惯、时间模式
[你的项目路径]/memory-system/Memory/L3_认知层.yaml       # 思维模式、决策框架
[你的项目路径]/memory-system/Memory/L4_核心层.yaml       # 核心价值观、决策原则
```

### Step 2: 脑倒空环节

引导用户把脑子里所有想做的事说出来：

```
现在进入【脑倒空环节】。

请把你脑子里所有想做的事、临时想法、新输入信息全部说出来。
不用筛选，不用排序，想到什么说什么。
我会记录下来，然后帮你整理。

你现在脑子里有哪些事？
```

**等待用户回复，不做打断，不做评判。**

### Step 3: 目标对齐

基于记忆系统，分析每个待办事项：

| 维度 | 说明 | 数据来源 |
|------|------|----------|
| 目标匹配度 | 与长期目标的关联程度 | Intent/目标与规划 |
| 重要性 | 对结果的影响程度 | L4 核心价值观 |
| 紧急性 | 时间敏感性 | 用户描述 |
| 精力要求 | 需要的注意力强度 | L2 时间模式 |

### Step 4: 智能排序

按以下公式计算优先级分数：

```
优先级分数 = 目标匹配度(40%) + 重要性(30%) + 紧急性(20%) + 精力匹配(10%)
```

**排序原则**（来自 L4 核心层）：
- 效率优先：优先选择投入产出比高的任务
- 结果导向：以最终产出为衡量标准

**时间块匹配**（来自 L2 行为层）：
- 早上：高精力任务（复杂思考、创造性工作）
- 下午：中等精力任务（沟通、协作、学习）
- 晚上：低精力任务（整理、浏览、创意探索）

### Step 5: 任务拆分

将任务分为两类：

#### 保护时间任务（必须亲自做）

特征：
- 需要深度思考、创意决策
- 涉及核心价值产出
- 需要人际关系处理
- 学习新知识/新技能

#### Agent自动任务（可委托AI）

特征：
- 重复性执行工作
- 信息收集、整理、汇总
- 代码生成、文档撰写
- 数据分析、报告生成

### Step 6: 生成执行表并保存

**必须执行的操作**：使用 Write 工具将规划保存到文件

**文件路径**：`[你的项目路径]/daily-records/今日规划_[当前日期].md`

**输出格式**：

```markdown
# 今日规划 - [日期]

created: [创建时间]
status: 进行中

## 一、今日目标

> 今日核心目标：[一句话]

## 二、任务清单

| 优先级 | 任务 | 分类 | 时间块 | 预计时长 |
|--------|------|------|--------|----------|
| S | [必做任务1] | 保护时间 | 早上 | 2h |
| S | [必做任务2] | 保护时间 | 早上 | 1.5h |
| A | [重要任务] | 保护时间 | 下午 | 1h |
| A | [Agent任务] | Agent自动 | 下午 | - |
| B | [可选任务] | 保护时间 | 晚上 | 30min |

## 三、时间块安排

### 早上（高精力时段）
- [ ] [任务1] - [具体行动]
- [ ] [任务2] - [具体行动]

### 下午（中等精力）
- [ ] [任务3] - [具体行动]
- 🤖 Agent: [Agent任务描述]

### 晚上（低精力）
- [ ] [任务4] - [具体行动]

## 四、Agent并行执行建议

可同时启动的Agent任务：
1. [Agent任务1] → 预计产出：[结果]
2. [Agent任务2] → 预计产出：[结果]

## 五、今日约束提醒

- [来自约束与边界的提醒]

## 六、脑倒空原始记录

> 用户原始表述，便于复盘时回顾

[用户说的原始内容]
```

### Step 7: 更新记忆系统

规划完成后，更新记忆系统：

#### 更新 L1 情境层

```yaml
# 添加到 [你的项目路径]/memory-system/Memory/L1_情境层.yaml

entries:
  - date: [今日日期]
    type: "每日规划"
    tasks_planned: [任务数量]
    key_focus: [今日核心目标]
```

## 模式二：今日复盘

### 触发方式
- 说「开始今日复盘」
- 说「/daily-review」
- 说「今日复盘」

### Step 1: 读取今日规划

**首先读取**：`[你的项目路径]/daily-records/今日规划_[当前日期].md`

如果文件不存在，提示用户先进行规划或手动输入今日任务。

### Step 2: 引导复盘提问

依次询问用户以下问题：

```
现在进入【今日复盘环节】。

我会问几个问题，帮你梳理今天的情况：

**问题 1/5**：今天实际完成了哪些任务？（对照早上的规划）
```

**问题清单**：

1. **完成情况**：今天实际完成了哪些任务？（对照规划）
2. **未完成原因**：有哪些任务没完成？原因是什么？
3. **意外发现**：今天有什么意外的收获或发现？
4. **情绪状态**：今天的精力状态如何？有什么情绪波动？
5. **明日预览**：明天有什么重要事项需要提前准备？

## 注意事项

1. **必须先加载记忆系统**，否则排序缺乏依据
2. **脑倒空环节不评判**，让用户充分表达
3. **保护时间任务不超过3个**，避免精力分散
4. **Agent任务可以并行**，提高效率
5. **规划文件必须保存**，便于复盘对照
6. **复盘要对照规划**，形成闭环
7. **更新记忆系统**，保持记忆的连续性
